Formal id: GreatestPrimeFactorLimit
Parameter Name: GreatestPrimeFactorLimit
Type: int
Category: Simulation Control
Initial Value: 2
Initial Unit: 
Description: Allows simulations with the number of the signal samples whose greatest prime factor (GPF) does not exceed the specified limit. For example, for 150 samples the GPF is equal to 5 (150 = 2x3x5x5). If the signal is resampled during simulation, the new sample rate may be automatically adjusted (increased) to satisfy the above condition. Holding the GPF limit is important to ensure high speed of the fast Fourier transform that is widely used in simulations. The highest speed is achieved with the default setting 2, which is equivalent to the condition that the number of samples is an integer power of two. It is recommended to set the GPF limit not larger than 13. Setting it to -1 will deactivate the GPF limiting, which allows to simulate an arbitrary number of samples in the signal.

Formal id: LogicalInformation
Parameter Name: LogicalInformation
Type: enumeration
Category: Simulation Control
Initial Value: ON
Initial Unit: 
Description: Logical information is generated by transmitters and used to reconstruct the transmitted data for BER estimation and clock recovery. If set to ON, logical information is generated and kept in memory for each simulation run. If set to CurrentRun, only the information for the current run is kept (data from previous runs is removed to reduce memory consumption). If set to OFF, logical information is not generated. LogicalInformation = OFF is particularly useful for Photonic Circuits simulations.
